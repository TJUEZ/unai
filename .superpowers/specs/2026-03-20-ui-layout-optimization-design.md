# UI 布局优化设计文档

## 概述

对 AIGC 检测工具的前端界面进行布局优化，移除重复按钮，统一控制元素位置，提升用户体验。

## 问题陈述

1. **重复的"开始检测"按钮**：左侧边栏有检测图标按钮，编辑面板头部也有"开始检测"按钮，功能重复
2. **不必要的"字数统计"按钮**：需要点击弹窗才能看到字数，应直接实时显示
3. **分块选择器位置不合理**：目前在右侧"检测结果"面板，应移至左侧"文档编辑"旁边

## 设计方案

### 布局结构（保持不变）

- 左侧边栏：图标工具栏
- 主区域：左（文档编辑）+ 右（检测结果），保持左右关系

### 具体改动

#### 1. 移除侧边栏检测按钮

**改动位置**：`App.tsx` 的 `ToolbarButton` 部分

- 移除侧边栏的 `IconSearch`（🔍检测）按钮
- 只保留编辑面板头部的"开始检测"按钮

#### 2. 编辑面板头部整合

**改动位置**：`App.tsx` 的 `editor-panel .panel-header` 部分

**改动后布局（从左到右）**：
```
[文档编辑] [字数: 1234 字] [分块选择器] [开始检测]           [AI概率条 45.2%]
```

- **字数显示**：直接展示文本字数（如 "1234 字"），去掉"字数统计"按钮
- **分块选择器**：从右侧移到此处，包含选项：原文、200字/块、500字/块、1000字/块
- **开始检测按钮**：唯一的检测入口
- **AI概率条**：仅在检测结果显示，位于最右侧

#### 3. 纯文本模式适配

**改动位置**：`PlainTextEditor` 组件

**改动后布局**：
```
[文档编辑] [字数: 1234 字] [分块选择器] [开始检测]
```

- 同样整合到编辑器头部
- 去掉"字数统计"按钮
- 实时显示字数

#### 4. 预览面板头部简化

**改动位置**：`App.tsx` 的 `preview-panel .panel-header` 部分

- 仅保留"检测结果"标题
- 分块选择器已移走
- AI概率条移至左侧编辑面板
